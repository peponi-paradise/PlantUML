1. Introduction



  1. Use case diagram이란, 시스템이 수행하는 작업에 대한 모델링을 의미한다.
    특히 High level 영역에서의 기능과 범위를 나타낸다.
  2. 사용자를 중심으로 시스템과의 상호작용을 요약하므로, 자세한 정보는 나타나지 않는다.
  3. 기본적인 다이어그램 그리기 방법은 Sequence diagram으로부터 출발한다.

2. 기본 사용법


  1. Actor 정의 : :이름: or actor 이름
  2. Use case 정의 : (이름) or usecase 이름

CodeRender 
@startuml

actor A
:B:

usecase C
(D)

A -> D
B -> C : Text

@enduml

3.3. 화살표 사용법


CodeRender 
@startuml

:A:

(a)
(b)
(c)

' `-` 대시가 많아질 수록 연결선이 길어짐
A -> a
A --> b
A ---> c

@enduml


@startuml

:A:
:B:

(a)
(b)

A -> a

' `.` 이용하여 모양 변경 가능
A .> b

' `-|>` 이용하여 확장의 의미로 사용 가능
A -|> B

@enduml


@startuml

:A:

(up)
(down)
(left)
(right)

a -up-> up
a -down-> down
a -left-> left
a -right-> right

@enduml


3.4. Package 사용법


CodeRender 
@startuml

package People{
    actor A
    actor B
}

rectangle Tasks{
    usecase a
    usecase b
}

A -> a
B -> b

@enduml


3.5. Note 사용법


CodeRender 
@startuml

actor A
actor B

note top of A : Top
note bottom of A : Bottom
note left of A : Left
note right of B : Right

note "Connected" as c

' Use `..` symbol for link objects
A..c
c..B

@enduml
